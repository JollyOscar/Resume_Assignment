Part 1: Job Description Analysis

To tailor my resume for the Irving Shipbuilding Systems Administrator position, I conducted a thorough analysis of the job posting to identify the most critical qualifications and high-impact phrases. The position emphasizes both technical competencies and operational methodologies, requiring a strategic approach to resume formatting.

Key Skills & Qualifications Identified

  • Core Infrastructure: Active Directory (AD), Windows Server, Linux Server OS, Virtualization
  • Methodologies: Agile framework, Industry leading practices, ITIL Foundations, Standardization
  • Operational Focus: Process automation, repeatability, system administration, stability of critical business solutions
  • Professional Requirements: Canadian Government Security clearance eligibility, CGP, ITAR, on-site presence

Target Keywords & Phrases

The job description repeatedly emphasized the following concepts, which I incorporated verbatim into my resume:

  • "Process automation to improve efficiency and repeatability"
  • "Technical leadership"
  • "Securing infrastructure"
  • "Stability of critical business solutions"
  • "Monitoring and analyzing capacity"
  • "Standardization"

Part 2: Formatting Choices & Design Rationale

Layout Structure: Functional Project-Based Design

I chose a two-column, functional project-based layout over a traditional chronological format for several strategic reasons:

Left Column (33% width):

  • Technical Skills Section: Positioned prominently to ensure ATS systems immediately identify core competencies like Active Directory, Windows Server 2025, and process automation
  • Education & Eligibility: Condensed format emphasizes current enrollment in IT Systems Management while addressing security clearance eligibility requirements
  • Dark Background (#333333): Creates visual hierarchy and draws attention to critical keywords without overwhelming the reader

Right Column (67% width):

  • Career Profile: Opens with a targeted summary incorporating key phrases from the job posting (process automation, securing infrastructure, standardization, technical leadership)
  • Core IT Projects: Replaces traditional "Work Experience" to shift focus from warehouse logistics to IT competencies, demonstrating practical application of required skills
  • Professional Strengths: Addresses non-technical requirements (on-site reliability, safety compliance, attention to detail) without emphasizing non-IT work history

Typography & Readability

I selected Roboto font for its clean, professional appearance and high readability in both screen and print formats. Font sizes were carefully calibrated:

  • Section headers (h2): 1.05rem, uppercase, with border-bottom dividers for clear section breaks
  • Body text: 0.9-1.0rem with line-height 1.35 for optimal density without cramping
  • Project titles: Bold, 1.0rem to establish hierarchy without dominating the page

Strategic Bold Text Usage

Throughout the resume, I bolded specific keywords that mirror the job posting to enhance ATS parsing and facilitate rapid scanning by hiring managers:

  • Technical Skills: "Active Directory (AD DS)", "Windows Server 2025", "Bash scripting"
  • Action Verbs in Projects: "Architected", "Configured", "Automated", "Deployed", "Developed", "Orchestrated"
  • Key Concepts: "Process automation", "repeatability", "standardization", "securing infrastructure"

A4 Print Optimization

The resume is designed to 210mm × 296mm A4 dimensions with print-specific CSS media queries to ensure:

  • All content fits on a single page (critical for ATS parsing and recruiter convenience)
  • Background colors print correctly using print-color-adjust: exact
  • Reduced margins (padding: 18-20px) maximize usable space while maintaining whitespace balance
  • GitHub repository links remain functional in digital versions while appearing professional in print

Part 4: Proofreading Process & Feedback Integration

Step 1: Comparative Keyword Audit

After drafting the resume, I conducted a line-by-line comparison between the job posting's "Responsibilities" section and my resume content. I created a checklist to verify that each major responsibility had a corresponding achievement or skill statement:

  • ✓ Process automation → "Automated deployments using custom Bash scripts"
  • ✓ Technical leadership → Career Profile opening sentence
  • ✓ Securing infrastructure → Repeated 3 times across Projects section
  • ✓ Stability of critical business solutions → Windows Server project
  • ✓ Standardization → Cloud Infrastructure project + Career Profile

Step 2: Peer Review Feedback

I shared my initial draft with a peer reviewer (fellow IT Systems Management student) who provided critical feedback:

Initial Feedback: "The resume feels 'academic' rather than 'professional.' It reads like a school project rather than enterprise-level work."

Specific Issues Identified:

  • Projects were titled generically ("Linux Project", "Windows Lab")
  • Bullet points used passive voice ("Configured DNS", "Set up AD")
  • Lacked industry-specific terminology from the job posting

Step 3: Revisions Based on Feedback

To address the "academic" tone, I implemented the following changes:

A. Reframed Project Titles with Business Context

  • Before: "Windows Server Project"
  • After: "Windows Server Infrastructure Design" (adds professional terminology)

B. Adopted Enterprise-Level Language

  • Before: "Configured Active Directory"
  • After: "Configured Active Directory (AD DS) for centralized authentication and implemented Group Policy Objects (GPOs) for security enforcement and securing infrastructure" (uses exact job posting phrase)

C. Replaced Passive Voice with Strong Action Verbs

  • Before: "DNS was configured using BIND9"
  • After: "Configured enterprise-level services including BIND9 DNS and Kea DHCP"

D. Added Strategic Keyword Repetition

Per the peer reviewer's suggestion, I ensured that the most critical job posting phrases appeared in at least 2-3 different sections to maximize ATS scoring:

  • "Securing infrastructure" → Career Profile, Windows Server project, Linux Systems project
  • "Process automation" → Career Profile, Technical Skills, Linux project, Professional Strengths
  • "Standardization" → Career Profile, Technical Skills, Cloud Infrastructure project

Step 4: Technical Proofreading

I used the following tools and techniques to ensure error-free submission:

  • Grammarly: Checked for spelling, grammar, and punctuation errors
  • Manual Print Preview: Verified that all content fits on one A4 page without cutting off
  • Link Testing: Confirmed that GitHub repository links and LinkedIn profile URL are functional
  • HTML Validation: Ensured proper character encoding for special symbols (&, –, etc.)

Step 5: Final Mentor Review

After implementing peer feedback, I shared the revised resume with a mentor (former IT professional) for a final assessment. Their feedback was positive:

"This revision is significantly stronger. The language now matches what we see in professional infrastructure roles. The 'Professional Strengths' section is a clever way to address the warehouse experience without making it the focus. Make sure the eligibility section emphasizes your willingness to pursue clearance—that's a common blocker for defense contractors."

Based on this feedback, I added the sentence: "Able to participate in Canadian Government security screening processes as required; safety-first mindset and compliance-focused work habits." This directly addresses the CGP/ITAR/clearance requirements without overpromising clearance that I don't yet possess.

Conclusion: Lessons Learned

Creating this targeted resume taught me that effective resume writing is less about listing every skill and more about strategic alignment with employer priorities. Key takeaways include:

  • Keyword placement matters: Using exact phrases from the job posting (not synonyms) improves ATS success rates
  • Format serves function: A functional project-based layout allowed me to emphasize IT skills over non-IT work history
  • Professional language elevates technical work: Reframing college projects with enterprise terminology (e.g., "securing infrastructure", "stability of critical business solutions") made my experience appear more relevant
  • Non-technical strengths matter: For roles requiring security clearances and on-site work, reliability and safety compliance can be differentiators

This process reinforced the importance of treating resume creation as an iterative, feedback-driven process rather than a one-time task. The peer and mentor reviews were invaluable in transforming an "academic" resume into a competitive professional document.
